Lab Network Design

Network Layout

The current lab uses VirtualBox networking to separate internet access from the internal host-only lab network. Kali Linux is used as the attacker system, while Ubuntu and vulnerable lab machines generate logs and security events that are monitored by Wazuh.

This design allows safe attack simulation, endpoint monitoring, alert investigation, and incident documentation without exposing intentionally vulnerable systems to the public internet.

Current Lab Environment

The active environment includes a Wazuh Manager, Kali Linux attack workstation, monitored Ubuntu endpoint, and Metasploitable2 target. The systems communicate through an isolated VirtualBox host-only network while selected machines use NAT when internet access is required.

Current Network: VirtualBox Host-Only Current Subnet: 192.168.56.0/24 Monitoring: Wazuh SIEM Attack Platform: Kali Linux

Workflow

SOC Investigation Process

Attack Simulation

Security events are generated from controlled lab activity such as failed SSH logins, privilege-escalation testing, account changes, reconnaissance, and vulnerable-service exploitation.

Detection and Alert Review

Wazuh alerts are reviewed to identify the affected endpoint, source IP address, rule level, MITRE tactic, event details, and supporting log evidence.

Incident Documentation

Findings are documented in professional incident reports containing an executive summary, event timeline, indicators of compromise, MITRE mapping, response actions, and lessons learned.

Portfolio Case Files

Completed investigations are converted into portfolio case files to demonstrate hands-on SOC skills, technical analysis, and professional security-documentation ability.
