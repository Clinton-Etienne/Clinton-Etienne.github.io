MEDIUM SEVERITY

IR-2026-001

Linux Privilege Escalation Investigation

Investigation of Linux administrative account creation, group creation, password assignment, sudo activity, and privilege escalation alerts generated by Wazuh SIEM.

Incident Overview

Status: Closed

Severity: Medium — Wazuh Rule Level 8

Date Detected: June 20, 2026 at 14:46:01

Analyst: Clinton Etienne

Affected Host: Ubuntu-Target — 192.168.56.107

Alert Source: Wazuh SIEM

Rule ID: 5901

Executive Summary

Wazuh generated Rule ID 5901 after detecting the creation of a new local group on the monitored Ubuntu-Target endpoint. The activity originated from the adduser process and was recorded through the system journal.

After reviewing the related alerts, the investigation determined that the activity was performed by an authorized administrator. No evidence of unauthorized access or malicious activity was identified.

Alert Details

Rule Name: New group added to system

Alert Description: New user group, user, and password created

Source IP: 192.168.56.107

Destination IP: 192.168.56.107

Username / Account: Clinton

MITRE ATT&CK Mapping

  • TA0004 — Privilege Escalation
  • TA0003 — Persistence
  • TA0005 — Defense Evasion
  • TA0001 — Initial Access

Timeline of Events

  1. User authenticated to Ubuntu-Target.
  2. First-time sudo usage was detected.
  3. Successful privilege escalation to root was observed.
  4. A new user account was created.
  5. A new local group was automatically created for the account.
  6. A password was assigned to the account.
  7. Wazuh generated Rule 5901 alert.

Investigation Findings

A review of related Wazuh alerts confirmed that the activity occurred during authorized administrative work. The account and group creation events were consistent with normal Linux administration.

No malicious behavior, persistence mechanism, unauthorized access attempt, or abnormal source activity was identified during the investigation.

Evidence

Wazuh alert evidence confirmed local account activity, group creation, password assignment, sudo usage, and privilege escalation-related events.

Wazuh screenshot evidence is included in the downloadable incident report.

Impact Assessment

No adverse impact was identified. The activity was performed by an authorized administrator account and did not indicate compromise.

Containment and Remediation

No containment actions were required because the activity was determined to be authorized administrative activity.

Recommended follow-up includes continued monitoring of sudo activity, account creation events, password changes, and administrative actions through Wazuh SIEM.

Lessons Learned

Not every security alert represents malicious activity. This investigation demonstrates the importance of validating administrative behavior before escalating an incident.

Proper log correlation and analyst review helped confirm that the activity was authorized while also proving that the monitoring controls were working as expected.

Final Disposition

Disposition: Authorized Activity / False Positive
